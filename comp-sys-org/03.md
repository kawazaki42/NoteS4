# Арифметические основы ЭВМ

Система счисления - система приемов наименования и записи чисел с помощью некоторого набора символов: __цифр__

числа состоят из нек-го набора символов (__цифр__)

макс. кол-во цифр называется __алфавитом__ системы счисления

Непозиционные системы
- римская
  - X
  - XX
  - XXX
  - XL
- египетская
- греческая
- недостатки:
  - сложно делать арифметику
- современное применени: в основном декоративное

Позиционные системы счисления: значение каждой (вес) цифры зависит от положения в числе (разряда)
- арабская
  - 1
  - 10
  - 100

Алфавит позиционной системы счисления ($s$) называется основанием системы счисления

Ну или макс. значение в разряде

Определяет макс. возможное кол-во чисел, записываемых в этой системе

$s$-ичная система счисления

Все позиционные системы счисления обладают аддитивно-мультипликативности.

Если по-русски, число можно представить в виде многочлена

```
x = e_r * s^r + e_{r-1} * s^{r-1} + ...
```

## Двоично-десятичные коды

Существуют поз. сис., которые одновременно используют несколько оснований

Это вспомогательная система

Может использоваться для реализации ввода-вывода

Или в компьютерах с десятичной арифметики (редкость)

Каждая десятичная цифра заменяется соот-щим 4-разрядным двоичным числом

Для записи цифр используется 2чная система счисления

Но вес каждой цифры (тетрады) остается десятичным

будет задача по BCD

если вдруг что - добиваем незначащими нулями

делаем overflow:
- 9 -> 9
- 10 -> 0
- 11 -> 1
- ...

## Формы представления чисел


В общем случае помимо цифр числа, необходимо хранить:
- его знак
- положение точки/запятой (разделителя целой и дробной части)

под знак обычно выделяется старший разряд (__знаковый__):
- 0 -> +
- 1 -> -

для указания положения точки используются две формы записи чисел:
- фиксированная
  - предполагает что все числа либо чисто целые, либо чисто дробные
  - в этом случае положение точки стандартно и не требует указания
  - для целых: точка зафиксирована справа от младшего разряда (а потом нули разумеется)
  - для дробных: точка зафиксирована слева от старшего разряда (а перед ней все нули)
  - недостатки:
    - не позволяет хранить смешанные числа (целое + дробь)
  - часто используется как вспомогательная форма
- плавающая (`float`)
  - более универсальна
  - $A = m_A * s^P$
  - число состоит из 2 частей:
    - мантисса ($m_A$)
      - хранит цифры числа
      - записывается как дробное число с фикс. точкой
    - порядок ($P$)
      - положение точки
      - записывается как целое с фикс. точкой
      - знак задает направление смещения точки
        - "+" -> вправо
        - "-" -> влево
  - знак числа равен знаку мантиссы
    - его обычно выносят в общий знаковый разряд
  - пример:
    - A = -123.456
      - m_A = -0.123456
      - P = +3
    - A = 0.00123456
      - m_A = 0.123456
      - P = -2
  - может быть нормализованным и ненормализованным
    - не нормализовано: если 1ая цифра мантиссы после запятой равна нулю
    - нормализовано: есла не равна
      - позволяет хранить больше значащих цифр
        - поэтому перед записью в память числа нормализуются
      - как нормализовать: двигаем число влево с уменьшением порядка
  - для увеличения точности при хранении мантиссы двоичных чисел
    - используется __способ скрытой единицы__
      - старший бит числа не хранится, т.к. он всегда равен 1
      - `P -= 1`
      - в осовободившийся разряд можно дописать еще одну цифру
      - перед арифметическими операциями старший бит надо восстановить
  - смещенный порядок
    - бывает пол. и отр.
    - для упрощения операций его часто смещают
    - $P_s = P + 2^(q-1)-1$
      - степень двойки минус один называется смещением
    - $q$ - кол-во разрядов порядка
    - это чтобы вывести операции в положительные числа
    - чтобы найти обычный порядок надо вычесть смещение
